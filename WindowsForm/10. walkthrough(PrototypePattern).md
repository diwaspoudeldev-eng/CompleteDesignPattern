# Walkthrough: Prototype Design Pattern

The **Prototype Design Pattern** has been implemented to allow for easy cloning of customer objects. This is useful when you want to create a new customer that is very similar to an existing one.

## How it was implemented

### 1. Interface Update
We added the `Clone()` method to the `ICustomer` interface.
**File**: [ICustomer.cs](file:///d:/Projects/Personal/Practice/CompleteDesignPattern/InterfaceLayer/ICustomer.cs)

### 2. Base Class Implementation
We implemented the `Clone()` method in `CustomerBase` using the built-in `.MemberwiseClone()` method, which performs a shallow copy of the object.
**File**: [MiddleLayer.cs](file:///d:/Projects/Personal/Practice/CompleteDesignPattern/MiddleLayer/MiddleLayer.cs)
```csharp
public virtual ICustomer Clone()
{
    return (ICustomer)this.MemberwiseClone();
}
```

### 3. UI Integration
A **"Clone & Add"** button was added to the form. When clicked, it takes the current active customer object, creates a clone of it, and immediately adds it to the **in-memory collection**.

**File**: [FormCustomer.cs](file:///d:/Projects/Personal/Practice/CompleteDesignPattern/WindowsForm/FormCustomer.cs)
```csharp
private void btnCloneAndSave_Click(object sender, EventArgs e)
{
    if (cust != null)
    {
        ICustomer clonedCust = cust.Clone();
        clonedCust.Id = 0; 
        _customers.Add(clonedCust); // Immediate Addition
        LoadGrid(); // See it in the grid!
    }
}
```

## When to use it
Use the Prototype pattern when the cost of creating a new object is expensive or when you want to create new objects by copying an existing "prototype" instance.

---
> [!TIP]
> In this implementation, `.MemberwiseClone()` creates a shallow copy. If your object had complex nested objects, you would need to implement a deep copy logic.
