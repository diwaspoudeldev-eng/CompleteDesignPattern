# Walkthrough: Iterator Design Pattern

This walkthrough explains the implementation of the **Iterator Design Pattern** in our C# project to handle in-memory data collections and batch saving.

## Concept Overview
The Iterator pattern provides a way to access the elements of an aggregate object (like a list) sequentially without exposing its underlying internal structure. 

In this project, we use it to manage a "pending" list of customers that can be reviewed in the UI before being committed to the database.

## Key Components

### 1. The Collection: `CustomerCollection`
We implemented a custom collection that holds `ICustomer` objects. By implementing `IEnumerable<ICustomer>`, we enable standard iteration (like `foreach`) for the UI layer.

**File**: [CustomerCollection.cs](file:///d:/Projects/Personal/Practice/CompleteDesignPattern/MiddleLayer/CustomerCollection.cs)
```csharp
public class CustomerCollection : IEnumerable<ICustomer>
{
    private List<ICustomer> _customers = new List<ICustomer>();

    public void Add(ICustomer customer) { _customers.Add(customer); }
    public void Clear() { _customers.Clear(); }

    public IEnumerator<ICustomer> GetEnumerator()
    {
        // Using yield return for the iterator implementation
        foreach (var customer in _customers)
        {
            yield return customer;
        }
    }
    // ...
}
```

### 2. The Iterator in UI Logic
The UI interacts with this collection through three distinct operations:

**File**: [FormCustomer.cs](file:///d:/Projects/Personal/Practice/CompleteDesignPattern/WindowsForm/FormCustomer.cs)

- **Normal Add (`btnAdd_Click`)**: Saves the current field data directly to the database.
- **Add (Iterator) (`btnAddInmemory_Click`)**: Adds the customer to our `CustomerCollection`. The Grid effectively shows a "Unified View" of both DB data and these in-memory items.
- **Batch Save After Iteration (`btnBatchSave_Click`)**: This is where the pattern shines. It iterates through the collection and saves everything.

```csharp
private void btnBatchSave_Click(object sender, EventArgs e)
{
    // The foreach loop uses the Iterator Pattern to traverse our custom collection
    foreach (var customer in _customers)
    {
        repo.Add(customer); // Persistence logic
    }
    _customers.Clear(); // Empty memory
    LoadGrid(); // Refresh from DB
}
```

## Benefits and Logic
1.  **Encapsulation**: The UI doesn't know (or care) that `CustomerCollection` uses a `List` internally.
2.  **Flexible UI**: We can display "pending" items in the grid alongside saved ones using `Concat` on the iterators.
3.  **Batch Processing**: It provides a clean, standard way to commit multiple records in a single user action.

---
> [!NOTE]
> This pattern adheres to the **Open/Closed Principle**, as we can change the internal storage of the collection without changing the UI code that iterates over it.
